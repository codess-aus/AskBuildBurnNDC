# Ask Build Burn Less - NDC Sydney Talk

> **The fastest teams are not the ones that type the most, they're the ones that reduce uncertainty early.**

## Welcome

This comprehensive guide is based on Michelle Sandford's talk at NDC Sydney about building better software, asking the right questions, and burning out less. Throughout these chapters, you'll learn practical tactics for:

- **Asking Better Questions** - Understanding what questions to ask, when to ask them, and how to ask in a way that improves quality
- **Building Better Software** - Writing less throwaway code, avoiding rework, and making decisions that last
- **Burning Out Less** - Creating clear boundaries, preventing context switching chaos, and eliminating 2 a.m. surprises
